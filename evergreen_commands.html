evergreen notes (commands)

  • The ME column indicates whether the ME will act on this command.
  • The PFP column indicates whether the PFP will treat this kind of command specially or just forward it to the ME verbatim.
  • The Linux column indicates whether the kernel module will allow userspace send these commands to the GPU. Linux means that the command name shows up in the header file evergreend.h but is still not allowed by evergreen_packet3_check.
  • The Mesa column shows whether the command is mentioned in its header files.
  • The prog. guide column lists commands that are documented in the "Evergreen Programming Guide"

(The PFP and ME are two components in AMD gpus interpreting commands.)

Versions

The PFP and ME columns are based on the firmware blobs PALM_me.bin and PALM_pfp.bin in linux-firmware commit 9c71af939b51ce1431087acfb5b0f61c0b491c63.

The Linux column is based on version 4.8.6.

The Mesa column is based on commit 3f5fba8a7be61bfc0f46a5ea058108f6e0e1c268.

The Programming Guide states to be Revision 1.0 of May 24, 2011. Its sha1 is a2cb6a64f12029a544b05589ad583a48a6e4b2d5.
